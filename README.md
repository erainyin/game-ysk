# YSK大作战 - 游戏规则与逻辑说明

## 游戏概述

YSK大作战是一款基于网页的多人掷骰子策略游戏。玩家通过掷骰子在棋盘上移动，躲避危险格子，利用有利格子加速前进，召唤幽灵辅助战斗，最终第一个到达终点或成为最后存活者获胜。

### 技术栈

- **前端框架**: 纯HTML5 + CSS3 + JavaScript (ES6+)
- **无第三方依赖**: 无需npm安装，直接打开index.html即可运行

***

## 游戏配置

### 基础配置 (`js/config.js`)

| 配置项             | 值                                             | 说明    |
| --------------- | --------------------------------------------- | ----- |
| `ROWS`          | 8                                             | 棋盘行数  |
| `COLS`          | 8                                             | 棋盘列数  |
| `MIN_PLAYERS`   | 2                                             | 最小玩家数 |
| `MAX_PLAYERS`   | 4                                             | 最大玩家数 |
| `PLAYER_COLORS` | \['#e74c3c', '#3498db', '#f39c12', '#9b59b6'] | 玩家颜色  |
| `PLAYER_NAMES`  | \['玩家1', '玩家2', '玩家3', '玩家4']                 | 玩家名称  |

### 棋盘规格

- **总格子数**: 8 × 8 = 64格
- **起点**: 第1格
- **终点**: 第64格
- **路径方向**: 蛇形路径，偶数行从左到右，奇数行从右到左

***

## 棋盘与路径

### 路径规则

棋盘采用蛇形路径设计：

```
[1]→[2]→[3]→[4]→[5]→[6]→[7]→[8]
                               ↓
[16]←[15]←[14]←[13]←[12]←[11]←[10]←[9]
 ↓
[17]→[18]→[19]→[20]→[21]→[22]→[23]→[24]
                               ↓
...以此类推...
```

### 危险区域

棋盘最后1/5区域（第52-64格）为危险区域，进入此区域后：

- 超过其他玩家会导致对方血量减1
- 危险区域格子有黄色边框标识（`#f39c12`，3px）

## 格子属性详解

游戏中有13种格子属性，定义在 `js/cell_properties.js` 的 `FUNCTION_MAP` 中：

### 血量变化 (BL)

- **标记格式**: `❤️X` 或 `🖤X`
- **说明**: X为变化量，正值为加血，负值为减血
- **示例**: `❤️+3` 表示加3滴血，`🖤-2` 表示减2滴血

### 加速前进 (FF)

- **标记格式**: `➡️X`
- **说明**: 额外向前移动X步
- **示例**: `➡️3` 表示额外前进3步

### 加速后退 (FB)

- **标记格式**: `⬅️X`
- **说明**: 向后退X步（不能退到起点之前）

### 超速前进 (FLF)

- **标记格式**: `🚀X`
- **说明**: 向前移动当前掷骰子数 × X 的步数
- **示例**: 掷骰子得5，踩到`🚀2`，则前进 5 × 2 = 10步

### 超速后退 (FLB)

- **标记格式**: `🚀X`
- **说明**: 向后退当前掷骰子数 × X 的步数
- **示例**: 掷骰子得5，踩到`🚀2`，则后退 5 × 2 = 10步

### 炸弹爆炸 (BB)

- **标记格式**: `💣X`
- **说明**: 以当前格子为中心，前后各X格范围内的所有玩家血量减1
- **动画**: 爆炸时显示 `assets/bb.gif`，尺寸放大到200%
- **示例**: `💣2` 表示影响范围为当前格子前后各2格

### 死亡陷阱 (DDD)

- **标记格式**: `💀`
- **说明**: 无变量，玩家踩到直接死亡
- **豁免**: 贴身幽灵可代替玩家死亡，不死守护状态下可免疫

### 不死守护 (UND)

- **标记格式**: `🛡️X`
- **说明**: X为不死回合数，期间踩到DDD可不死
- **生效时间**: 从下一回合开始计算

### 顺序变换 (CR)

- **标记格式**: `🔀`
- **说明**: 无变量，玩家顺序反转
- **示例**: 顺序 `1→2→3` 变为 `3→2→1`

### 黑洞 (BH)

- **标记格式**: `🕳️X`
- **说明**: X为黑洞编号，踩到后移动到当前骰子数对应的黑洞
- **特殊机制**: 黑洞位置在加载时随机分配（最多6个），不在CSV中定义

### 格子跳转 (TO)

- **标记格式**: `🌀X`
- **说明**: X为目标格子编号，直接跳转到对应格子

### 暂停 (P)

- **标记格式**: `⏸️X`
- **说明**: 玩家将跳过X个回合，暂停期间无法掷骰子
- **示例**: `⏸️2` 表示跳过2个回合

### 幽灵召唤 (GST)

- **标记格式**: `👻`
- **说明**: 踩到后可召唤幽灵（普通或贴身）

***

## 幽灵系统

### 幽灵类型

| 类型   | 编号 | 特点                              |
| ---- | -- | ------------------------------- |
| 普通幽灵 | 1  | 独立移动，速度为骰子数的2倍，初始位置为起点          |
| 贴身幽灵 | 2  | 跟随玩家移动，可代替玩家抵挡DDD死亡，初始位置为玩家当前位置 |

### 幽灵召唤规则

1. 每个玩家最多拥有1个幽灵
2. 召唤相同类型的幽灵会增加血量（最多3滴）
3. 召唤不同类型的幽灵会替换原有幽灵，血量重置为1
4. 幽灵血量上限为3滴（在 `player.js` 的 `changeGhostHealth` 方法中限制）

### 幽灵操作

- **普通幽灵**: 掷骰子后可选择移动玩家或幽灵
- **贴身幽灵**: 自动跟随玩家移动，无法单独操作
- **幽灵移动**: 速度为玩家的2倍（骰子数 × 2）

### 幽灵死亡

- 幽灵血量减至0时死亡消失
- 幽灵死亡不影响玩家
- 贴身幽灵死亡后不再提供DDD保护

## 游戏流程

### 游戏状态

| 状态        | 说明     |
| --------- | ------ |
| `waiting` | 等待游戏开始 |
| `playing` | 游戏进行中  |
| `ended`   | 游戏结束   |

### 回合规则

1. 每个玩家每回合只能掷一次骰子
2. 掷骰子后根据点数移动
3. 移动到目标格子后触发格子属性
4. 完成所有操作后切换到下一玩家

### 回合切换

- 按玩家顺序轮流进行
- 跳过已获胜、已死亡、暂停中的玩家
- 当所有玩家都完成一回合后，回合数加1

### 暂停机制

- 踩到暂停格子（P+X）会暂停X个回合
- 暂停期间无法掷骰子，但位置保持不变

***

## 胜负判定

### 胜利条件

1. **到达终点**: 第一个到达最后一格（第64格）的玩家立即获胜，游戏结束
2. **最后存活**: 当只剩一个玩家存活时，该玩家获胜，游戏结束

### AI模式特殊规则

在人机对战模式下：

- 人类玩家（`humanPlayerIndex`）死亡后，游戏立即结束，显示"你输了"
- AI玩家死亡不影响游戏继续，直到只剩人类玩家或人类玩家死亡

### 游戏结束判断顺序 (`game.js` 的 `checkGameEnd` 方法)

1. 检查是否有玩家到达终点（立即获胜）
2. 检查存活玩家数量（只剩1人则获胜）
3. 检查AI模式下人类玩家是否死亡（立即结束）
4. 否则继续下一轮

***

## 人机对战模式

### 模式切换

- 在玩家选择弹窗中勾选"人机大战"选项
- 人类玩家控制一个角色，其余为AI控制

### AI行为

- AI自动掷骰子和移动
- AI掷骰子后400ms内自动选择幽灵类型（随机选择普通或贴身）
- AI自动选择移动目标（玩家或幽灵，50%概率）

### UI限制

- AI模式下，非人类玩家的骰子按钮被禁用
- 只有人类玩家回合时骰子按钮可用

## 地图系统

### 地图文件格式

地图使用CSV文件存储，包含以下列：

| 列名     | 说明               |
| ------ | ---------------- |
| `格子`   | 格子编号（1-64）       |
| `功能`   | 功能标识（如BL、FF、BB等） |
| `变量`   | 功能参数（如+3、-2、#1等） |
| `中文名称` | 功能的中文名称          |
| `规则`   | 功能的详细说明          |
| `英文全称` | 功能的英文名称          |

### 地图文件

| 文件                            | 说明       |
| ----------------------------- | -------- |
| `grid.csv`                    | 默认地图     |
| `map/Map1.csv`                | 地图1      |
| `map/Map2-noGST.csv`          | 无幽灵召唤的地图 |
| `map/Map3-easy4newplayer.csv` | 新手简单地图   |
| `map/Map4-hard.csv`           | 困难地图     |

### 地图加载流程

1. 点击"选择地图"按钮选择CSV文件
2. 解析CSV内容并生成格子属性
3. 随机分配黑洞位置（最多6个）
4. 渲染棋盘

## 技术架构

### 核心类

| 类        | 文件             | 职责                 |
| -------- | -------------- | ------------------ |
| `Game`   | `js/game.js`   | 游戏主逻辑，管理玩家、回合、胜负判定 |
| `Player` | `js/player.js` | 玩家状态管理，位置、血量、幽灵    |
| `Board`  | `js/board.js`  | 棋盘管理，路径计算、格子定位     |
| `Dice`   | `js/dice.js`   | 骰子逻辑，掷骰子动画和结果      |
| `UI`     | `js/ui.js`     | 用户界面，渲染、交互、事件处理    |

### 事件回调

`Game` 类通过回调机制与 `UI` 类通信：

- `onStateChange`: 游戏状态变更
- `onPlayerMove`: 玩家移动
- `onDiceRoll`: 骰子结果
- `onGameEnd`: 游戏结束
- `onNotification`: 通知消息
- `onLog`: 游戏日志
- `onBombExplode`: 炸弹爆炸动画
- `onGhostSelect`: 幽灵选择
- `onMoveSelect`: 移动目标选择

***

## UI/UX特性

### 响应式设计

- **桌面端**: 完整功能按钮显示
- **移动端（≤768px）**:
  - 按钮变为40×40固定尺寸
  - 使用emoji代替文字：▶️开始游戏、🔄重新开始、✍️地图编辑、🗺️选择地图
  - 玩家列表只显示当前玩家完整信息，其他玩家显示颜色圆点

### 游戏日志

- 日志按回合分组显示
- 每个回合下按玩家顺序显示操作记录
- 最新回合显示在最上方

### 提示信息

- 悬停格子显示属性说明
- 相关格子高亮显示（如炸弹范围、跳转目标）
- 实时通知玩家操作结果

### 重新开始

- 点击重新开始会弹出确认对话框
- 确认后清空日志并显示开始游戏弹窗
